# Investment Masters —— AI 投资大师委员会

一个 OpenCalw Skill，召唤五位顶级投资大师组成虚拟委员会，对任意标的进行独立分析和平权投票，输出可执行的投资裁决报告。

## 为什么做这个？

投资决策最怕单一视角盲区。这个系统把五种截然不同的投资哲学——价值投资、周期研判、本分哲学、宏观交易、量化分析——强制并行运行，互相看不到对方的结论，最后用规则整合成一份裁决。

本质上是用 AI 模拟一个高质量的投资委员会流程。

## 五位投资大师

| 委员 | 核心视角 | 擅长捕捉 |
|------|---------|---------|
| **查理·芒格** | 护城河 / 生意本质 / 能力圈 | 长期竞争优势、管理层质量、估值安全边际 |
| **霍华德·马克斯** | 周期位置 / 情绪温度 / 第二层思维 | 市场过热/过冷信号、风险不对称性 |
| **段永平** | 本分 / 动机审查 / 十年持有 | 生意是否真正理解、买入动机是否纯粹 |
| **德鲁肯米勒** | 宏观流动性 / 催化剂 / 止损纪律 | 宏观拐点、流动性变化、交易时机 |
| **西蒙斯** | 量化五信号 / 动量 / 统计胜率 | 技术面异常、量价背离、统计套利机会 |

五位大师平权（各 20%），独立分析，互不可见。

## 三种分析模式

**买入判断** —— "分析腾讯"、"要不要买英伟达"、"帮我看看 BTC"

**仓位管理** —— "我持有半仓 BTC，怎么操作"、"已持有 NVDA，加仓还是减仓"

**比较分析** —— "腾讯和谷歌选哪个"

## 使用方法

### 触发

在 OpenCalw 对话中直接说：

```
投资大师怎么看英伟达
委员会分析腾讯
投资大师腾讯和谷歌选哪个
```

Skill 会自动匹配并启动分析流程。

### 支持的资产类型

- **股票**：A 股、港股、美股
- **ETF**
- **加密资产**：BTC 等
- **大宗商品**：黄金等

## 执行流程

```
数据准备 → 识别模式 → 并行召唤五位委员 → 整合裁决 → 输出报告
```

1. **数据准备**：通过 `scripts/fetch_price.py` 抓取实时行情（数据源：stooq.com），再用搜索补充基本面数据
2. **识别模式**：根据用户请求自动判断是买入判断、仓位管理还是比较分析
3. **并行分析**：同时启动 5 个独立 sub-agent，各自按对应大师的框架输出评分和判断
4. **整合裁决**：汇总五份独立意见，计算加权综合分，检查否决规则，输出最终建议

## 评级体系

| 综合得分 | 评级 | 建议仓位 |
|---------|------|---------|
| 8.0 – 10.0 | ⭐⭐⭐⭐⭐ 强烈买入 | 15–25% |
| 6.5 – 7.9 | ⭐⭐⭐⭐ 买入（保守） | 8–15% |
| 5.0 – 6.4 | ⭐⭐⭐ 等待/观望 | 0–5% |
| 3.5 – 4.9 | ⭐⭐ 谨慎 | 0% |
| < 3.5 | ⭐ 回避 | 0% |

### 否决规则

- **硬否决**：≥ 3 位委员评分 ≤ 4 → 直接回避，0% 仓位
- **软否决**（可叠加）：
  - 马克斯评分 ≤ 5 且市场情绪 = 贪婪 → 建议仓位减半
  - 段永平动机审查不通过 → 仓位上限 30%
  - 西蒙斯量化看空 → 减仓 20–30%

## 特殊资产说明

**加密资产 / 黄金**：芒格和段永平的框架偏向"无内在价值"给低分，属于体系特性而非 bug。报告会标注宏观/量化视角（马克斯、德鲁肯米勒、西蒙斯）更具参考价值。

## 设计理念

- **独立性**：五个 sub-agent 并行运行，互相不可见结论，避免群体思维
- **多样性**：价值派、周期派、本分派、宏观派、量化派——五种正交视角
- **可执行**：输出不是"建议关注"这种废话，而是具体的仓位比例、止损价、加仓条件
- **有纪律**：否决规则是硬编码的，不会因为情绪被覆盖
